Types implementing this trait require initialization of their storage contents after allocation before they can be used.

Example

For example types like Value have uninitialized associated storage. Accessing a newly allocated instance of Value would result in a panic or even undefined behaviour. To circumvent this it is required to initialize its associated contract storage via initialize.

fn default_value() -> Option<Self::Args>

The default value for default initialization purposes.

Returns None by default which means that the type does not support default initialization.

Note

Should be manually implemented only by those storage types that have a meaningful default initialization.

Example

The storage::Vec is an example for which it makes sense to implement this to create empty storage::Vec by default that have their length field initialized to be 0.

fn try_default_initialize(&mut self)

Tries to default initialize self.

Note

  • Does nothing if self does not support default initialization.
  • This should never be manually implemented by any implementer.
